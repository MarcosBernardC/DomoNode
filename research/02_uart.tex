\section{Comunicación: Requerimientos de Telemetría}
Para la evolución del ecosistema \textbf{DomoNode}, se requiere establecer un canal de comunicación asíncrono robusto que permita el envío de tramas de sensores hacia una etapa de agregación. El reto técnico reside en mantener la integridad de los datos operando a una frecuencia de oscilación baja (\SI{500}{\kilo\hertz}) para maximizar la eficiencia energética.

\section{Planificación y Modelado}
En esta segunda iteración, el enfoque se desplaza de la señal de diagnóstico (\textit{Heartbeat}) hacia la \textbf{Capa de Enlace}. Se utiliza el generador de baudios de 16 bits para mitigar errores de cuantización.

\subsection{Análisis de Error ($F_{OSC} = \SI{500}{\kilo\hertz}$)}
Utilizando el modo \texttt{BRG16=1}, se valida matemáticamente la estabilidad a \SI{9600}{bps}:
\begin{equation}
BaudRate = \frac{\SI{500000}{}}{4 \cdot (12 + 1)} = \SI{9615.38}{bps} \implies \text{Error: } \SI{0.16}{\percent}
\end{equation}

\subsection{Asignación de Recursos de Hardware}

\section{Construcción y Validación en Protoboard}
Implementación física para corroborar el análisis matemático previo.

\subsection{Esquemático de Conexión Serial}
Se integra un conversor USB-TTL al nodo \textbf{DomoNode}. Se mantiene el LED de \textit{Heartbeat} en \textbf{RD7} como monitor de estado del sistema durante la transmisión de ráfagas UART.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.5\textwidth]{DomoNode_squematic_inkscape.pdf}
    \caption{Diagrama de interconexión para validación de telemetría UART.}
    \label{fig:esquema_uart}
\end{figure}

\subsection{Pruebas de Campo e Integración}
El montaje en protoboard valida la ráfaga de datos cada \SI{60}{\second}. Se observa que a \SI{9600}{bps}, el tiempo de ocupación del bus es mínimo (\SI{12.5}{\milli\second}), lo que permite al \textbf{PIC16F1939} retornar a estados de bajo consumo rápidamente.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.5\textwidth]{research/assets/uartBcharProtoboardTest.jpg}
    \caption{Prototipo físico DomoNode V2: Pruebas de comunicación serial.}
    \label{fig:protoboard_v2}
\end{figure}

\subsection{Resultados del Monitoreo Digital}
La validación del flujo de datos se realizó mediante el software de monitoreo industrial \textbf{HTerm} sobre la distribución \textbf{Fedora}. Se confirmó la recepción íntegra del testigo de transmisión (\textit{testigo 'B'}) con una periodicidad de \SI{1}{\second}.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.7\textwidth]{research/assets/uartBcharHtermTest.png}
    \caption{Monitoreo multiformato en HTerm: Validación del testigo de transmisión. Se observa la consistencia entre el carácter ASCII 'B', su valor hexadecimal (0x42) y su representación binaria, confirmando la estabilidad del baudaje a \SI{500}{\kilo\hertz}.}
    \label{fig:hterm_success}
\end{figure}

\section{Conclusiones de la Iteración 02}
La validación física confirma que la configuración de 16 bits del BRG permite una comunicación fiable incluso a frecuencias de reloj reducidas. 
\begin{itemize}
    \item \textbf{Resultado:} Error de baudaje dentro de los límites industriales ($<\SI{2}{\percent}$).
    \item \textbf{Próximo Paso:} Integración del sensor de temperatura/humedad para generar tramas de datos reales en la Iteración 03.
\end{itemize}
