\section{Comunicación: Requerimientos y Alcance}
Para el nodo \textbf{DomoNode}, se ha identificado la necesidad de una señal de diagnóstico de bajo nivel (\textit{Heartbeat}). El sistema requiere una validación visual inmediata tras un evento de \textit{Master Clear} (MCLR) que confirme la estabilidad del oscilador interno antes de iniciar procesos de mayor complejidad.

\section{Planificación Rápida}
En esta etapa del \textbf{\faseactual}, el sprint de diseño se centra en la \textbf{Capa 0 (Hardware Core)}, estableciendo los cimientos para las capas superiores del modelo OSI mediante los siguientes objetivos:
\begin{itemize}
    \setlength{\itemsep}{2pt}
    \item Selección de un pin de salida dedicado exclusivamente a diagnóstico visual.
    \item Configuración del oscilador interno (\texttt{INTOSC}) para minimizar el BOM (\textit{Bill of Materials}).
    \item Definición de una secuencia de inicio que diferencie el estado de arranque de la ejecución normal.
\end{itemize}

\section{Modelado y Diseño Rápido}
Diseño orientado al microcontrolador \textbf{PIC16F1939}. El enfoque iterativo permite definir constantes físicas reutilizables en versiones superiores.
Este diseño se estructura desde la configuración del silicio (\textbf{Capa 0}), garantizando que los registros fundamentales (Fuses y Clock Tree) operen dentro de los márgenes nominales antes de la ejecución del firmware de aplicación.

\subsection{Capa 0: Configuración de Base de Tiempo}
Se selecciona el bloque \textbf{MFINTOSC} a \SI{500}{\kilo\hertz}. El modelado de tiempos en el nivel de registros se rige por:
\begin{equation}
F_{CY} = \frac{F_{OSC}}{4} \implies \frac{\SI{500}{\kilo\hertz}}{4} = \SI{125}{\kilo\hertz} \implies T_{CY} = \SI{8}{\micro\second}
\end{equation}

\section{Construcción del Prototipo}
Implementación física y carga de firmware.

\subsection{Firmware (XC8)}
Se implementa la lógica de control en C utilizando el compilador XC8. El código establece el \textit{Heartbeat} mediante retardos bloqueantes de \SI{500}{\milli\second}, precedidos por una ráfaga de inicio para validación de \textit{Power-on Reset}. El firmware final para producción utiliza la configuración \texttt{MFINTOSC} a \SI{500}{\kilo\hertz} para optimizar la eficiencia energética del nodo.

\subsection{Montaje en Protoboard}
El circuito fue ensamblado sobre protoboard siguiendo el esquemático de la Figura~\ref{fig:esquema_v1}. La alimentación se suministró desde el bus de \SI{5}{\volt} del programador PICKit2 vía ICSP, sin fuente externa adicional en esta iteración.

\section{Despliegue y Entrega (Pruebas)}

\subsection{Entorno de Simulación y Ajustes Técnicos}
Para la validación funcional se utilizó \textbf{PICSimLab v0.9.2} emulando una placa \textit{McLab2}. Durante las pruebas iniciales, se identificó una limitación en el paso de tiempo del simulador al operar por debajo de \SI{1}{\mega\hertz}. Para garantizar una correlación 1:1 con el tiempo real en el osciloscopio virtual, se optó por una \textbf{estrategia de ajuste de validación}:
\begin{itemize}
    \setlength{\itemsep}{2pt}
    \item Se reconfiguró el registro \texttt{OSCCON} a \texttt{0x30} (\SI{1}{\mega\hertz}).
    \item Se actualizó la macro \texttt{\_XTAL\_FREQ} a \SI{1}{\mega\hertz} para resincronizar los retardos.
\end{itemize}

\subsection{Resultados en Osciloscopio}
Tras el ajuste, el análisis de señales (Figura~\ref{fig:osciloscopio}) confirmó la precisión del firmware:
\begin{itemize}
    \setlength{\itemsep}{2pt}
    \item \textbf{Ciclo Completo:} \SI{1000.000}{\milli\second} (Exactitud del 100\% en simulación).
    \item \textbf{Duty Cycle:} 50\%, con semiciclos de \SI{500}{\milli\second}.
\end{itemize}

\section{Retroalimentación (Feedback)}
La Iteración 01 valida con éxito la estabilidad de la \textbf{Capa 0}. Como puntos clave para la Iteración 02 se definen:
\begin{itemize}
    \setlength{\itemsep}{2pt}
    \item \textbf{Optimización Energética:} Retornar a la frecuencia de \SI{500}{\kilo\hertz} en el prototipo físico para reducir el consumo de corriente dinámico.
    \item \textbf{Escalabilidad:} Implementar el manejo del LCD para mostrar estados de red, utilizando los tiempos de instrucción ya validados en esta fase.
\end{itemize}
